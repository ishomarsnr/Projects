SQL Filters and Linux File Permissions

Introduction

As part of my Google Cybersecurity course, I worked on projects to enhance the security of an organization's system by investigating potential security issues and updating file permissions. This involved using SQL queries with filters and Linux commands to retrieve specific information and set appropriate permissions.

Applying Filters to SQL Queries

Retrieve After Hours Failed Login Attempts

To investigate a potential security incident that occurred after business hours (after 18:00), I created a SQL query to filter for failed login attempts that occurred after business hours. 


-- Retrieve after hours failed login attempts
SELECT *
FROM log_in_attempts
WHERE login_time > '18:00'
  AND success = FALSE;

Retrieve Login Attempts on Specific Dates

To investigate a suspicious event that occurred on 2022-05-09, I created a SQL query to filter for login attempts that occurred on 2022-05-09 or on the day before. 


-- Retrieve login attempts on specific dates
SELECT *
FROM log_in_attempts
WHERE login_date = '2022-05-09'
   OR login_date = '2022-05-08';

Retrieve Login Attempts Outside of Mexico

To investigate login attempts that occurred outside of Mexico, I created a SQL query to filter for login attempts that occurred in countries other than Mexico. 


-- Retrieve login attempts outside of Mexico
SELECT *
FROM log_in_attempts
WHERE country NOT LIKE 'MEX%';

Retrieve Employees in Marketing

To update the computers for employees in the Marketing department in the East building, I created a SQL query to filter for these employees. 


-- Retrieve employees in Marketing
SELECT *
FROM employees
WHERE department = 'Marketing'
  AND office LIKE 'East%';

Retrieve Employees in Finance or Sales

To update the computers for employees in the Finance and Sales departments, I created a SQL query to filter for these employees. 


-- Retrieve employees in Finance or Sales
SELECT *
FROM employees
WHERE department = 'Finance'
   OR department = 'Sales';

Retrieve All Employees Not in IT

To update the computers for employees who are not in the Information Technology department, I created a SQL query to filter for these employees. 


-- Retrieve all employees not in IT
SELECT *
FROM employees
WHERE department != 'Information Technology';

File Permissions in Linux

Check File and Directory Details

To determine the existing permissions set for a specific directory in the file system, I used the `ls -la` command. 


# Check file and directory details
ls -la /path/to/projects

Describe the Permissions String

The 10-character string represents the permissions set on each file or directory. It can be deconstructed to determine who is authorized to access the file and their specific permissions. 


-rw-rw-r--
^ ^  ^  ^ 
| |  |  |
| |  |  └── Others (read-only)
| |  └──── Group (read/write)
| └─────── User (read/write)
└───────── File type (regular file)

Change File Permissions

To remove write permissions for others from the `project_k.txt` file, I used the `chmod` command. 


# Remove write permissions for others
chmod o-w project_k.txt
ls -la project_k.txt

Change File Permissions on a Hidden File

To remove write permissions from the user and group and add read permissions to the group for the hidden file `.project_x.txt`, I used the `chmod` command. 


# Change permissions on a hidden file
chmod u-w .project_x.txt
chmod g-w .project_x.txt
chmod g+r .project_x.txt
ls -la .project_x.txt
